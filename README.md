# 預貯金照会結果 検索ツール（file_search）GUI版 機能要件書

## 1. ツールの目的

共有ネットワーク上に保存された預金照会結果ファイル（.xlsm）を、宛名番号で素早く検索し、内容確認から差押予定一覧への追加までをシームレスに行うための補助ツール。deposit_seizure_list.exe との連携により、「ファイルを探す → 内容を確認する → 差押リストに追加する」という一連の業務フローを1つのツール内で完結させる。

本書はWPF GUI版（.cs/.exe）の機能要件を記述する。コンソール版（.ps1）の要件書も併せて参照のこと。

## 2. 技術構成

### 2.1 ビルド環境
- .NET Framework 4.0 / csc.exe による単一ファイルコンパイル。
- WPF（PresentationFramework, PresentationCore, WindowsBase, System.Xaml）を使用。
- XAML はインライン文字列として C# 内に埋め込み、XamlReader.Load で動的にパースする。

## 3. 起動方式

- file_search.exe をダブルクリック（または build.bat → 生成された exe を実行）で起動する。
- 起動時にウィンドウタイトルを「預貯金照会結果 検索ツール」に設定する。
- 引数は不要。
- 起動後、検索ボックスに自動フォーカスする。

## 5. 検索ボックス

### 5.1 構成
- 虫眼鏡アイコン（左端、#999）+ TextBox（Consolas 13px）+ × クリアボタン（右端）。
- プレースホルダ「宛名番号を入力...」（#999、テキスト入力時に非表示）。
- 右隣に「検索」ボタン（AB スタイル、虫眼鏡アイコン付き）。

### 5.2 × クリアボタン
- テキストがあるときのみ表示（TextChanged で Visibility を切替）。
- 丸背景（20px, CornerRadius=10）に × アイコン（Line 2本）。
- 通常: 背景 #E0E0E0 / アイコン #777。ホバー: 背景 #CCC / アイコン #555。
- Focusable=False（クリックしても検索ボックスのフォーカスが外れない）。
- クリック時の動作：テキストクリア → 検索結果リセット → フッタークリア（色も #666 に復帰）→ ボタン無効化 → 検索ボックスにフォーカス戻し。

### 5.3 検索実行
- Enter キーまたは「検索」ボタンクリックで ExecuteSearch を呼び出す。
- 空文字列の場合は無視する。

## 6. 検索結果テーブル

### 6.1 テーブル構造
- 「≡ 検索結果」セクションヘッダー（13px / #005FB8 / Medium、#FAFAFA 背景バー）。
- ListView + GridView（4列）。Background='White'。
- ItemContainerStyle に ResultItem スタイルを適用（アクセントバー + ホバー/選択）。

### 6.2 列定義
| 列 | 初期幅 | ヘッダー配置 | データ配置 | フォント | 備考 |
|----|--------|------------|----------|---------|------|
| ファイル名 | 240px（可変） | 中央 | 左寄せ（Margin 12,0,4,0） | Meiryo UI | TextTrimming + ToolTip |
| フォルダ | 180px | 中央 | 左寄せ（Margin 12,0,4,0） | Meiryo UI | TextTrimming + ToolTip |
| 更新日 | 130px | 中央 | 中央 | Consolas | MM/dd HH:mm 形式 |
| サイズ | 80px | 中央 | 中央 | Meiryo UI | KB 単位表示 |

### 6.3 列ヘッダー
- GridViewColumnHeader の暗黙的スタイル（キーなし TargetType）で全列に適用。
- フラットデザイン：#F5F7FA 背景、#777 テキスト、11px、中央揃え。
- ControlTemplate でデフォルトの立体感を除去。
- パディング列ヘッダー（右端の自動生成ヘッダー）にも同スタイルが適用される。

### 6.4 ListViewItem（ResultItem スタイル）
- Grid 重ね合わせ方式：rowBd（背景 Border）+ accent（3px 左端アクセントバー）。
- 通常: 白背景、下線 #F0F0F0。
- ホバー: #F0F4F8 背景。
- 選択: #E8F0FE 背景 + #005FB8 アクセントバー。
- GridViewRowPresenter Margin='0,7,0,7'（行パディング、deposit_seizure_list と統一）。

### 6.5 ソート
- 列ヘッダークリックでソート切替（昇順 ▲ / 降順 ▼）。
- デフォルト：更新日の降順（最新ファイルが先頭）。
- ソートインジケータ（▲/▼）をヘッダーテキストに付与。
- HeaderContainerStyle の Tag プロパティでソート列を識別（BasedOn で暗黙的スタイルを継承）。

### 6.6 列幅の自動調整
- ウィンドウリサイズ時に SizeChanged イベントで AdjustColumnWidths を呼び出す。
- フォルダ（180）・更新日（130）・サイズ（80）は固定幅、ファイル名が残幅を埋める（最小 150px）。

## 9. 履歴機能

### 9.1 検索履歴
- 検索実行時に履歴に追加（重複除去 → 先頭挿入 → 上限超過分を削除）。
- 上限：config.SearchHistoryMax（既定 5、JSON で変更可）。

### 9.2 ファイル履歴
- ファイルオープン時に履歴に追加（重複除去 → 先頭挿入 → 上限超過分を削除）。
- パスとオープン日時（DateTime）のペアで保存。
- 上限：config.FileHistoryMax（既定 10、JSON で変更可）。

### 9.3 永続化
- exe と同じフォルダの `history/` サブフォルダに、`search_{USERNAME}.json` を自動作成する。
- 検索履歴・ファイル履歴を1つの JSON ファイルに保存する。
- 検索実行時・ファイルオープン時に都度保存する。
- ファイル履歴の復元時は、存在するファイルのみを対象とする（削除済みファイルは自動除外）。
- 読み込みまたは書き込みに失敗した場合は、警告を1回表示し、以降はセッション内のみのメモリ保持に切り替える。

### 9.4 削除操作
| 操作 | 動作 |
|------|------|
| Delete キー | アクティブタブの選択中項目を削除 |
| 右クリック「削除」 | アクティブタブの選択中項目を削除 |
| 右クリック「全て削除」 | アクティブタブの履歴を全件クリア |

## 10. ファイル操作

### 10.1 ファイルを開く
- `Process.Start` で OS の既定アプリケーション（Excel）で開く。`UseShellExecute = true`。
- オープン前に `File.Exists` で存在チェック（不在時は MessageBox 警告）。
- オープン成功後にファイル履歴に追加し、フッター左にファイル名を表示。
- 検索結果とファイル履歴の両方から実行可能（共通の OpenFile メソッドに合流）。

### 10.2 フォルダを開く
- `explorer.exe /select,{fullPath}` で親フォルダを Explorer で開き、該当ファイルを選択状態にする。
- コンテキストメニューから実行可能。

### 10.3 複数選択
- 検索結果テーブルは SelectionMode='Extended'（Ctrl/Shift による複数選択対応）。
- 複数選択時は一括でファイルを開く / 差押リストに追加が可能。

## 11. deposit_seizure_list.exe との連携

### 11.1 呼び出し方式
- `Process.Start` で deposit_seizure_list.exe を別プロセスとして起動する。
- 引数にファイルパスを渡す（複数ファイルの場合はスペース区切り、各パスをダブルクォートで囲む）。
- `UseShellExecute = true`。

### 11.2 設定
- file_search_config.json の depositSeizureListScript に .exe のパスを設定する。
- 未設定または該当ファイルなしの場合は、「差押リストに追加」ボタンが非表示となり、検索とファイルを開く操作のみ可能。

### 11.3 エラーハンドリング
- ファイルが見つからない場合：MessageBox 警告。
- プロセス起動失敗：MessageBox エラー。

## 13. アニメーション

### 13.1 検索結果フェードイン
- 検索実行時に ListView 内の ItemsPresenter（データ行のみ、列ヘッダーは含まない）を対象とする。
- Opacity 0 → ForceRender → 検索処理 → DoubleAnimation(0→1, 100ms) でフェードイン。
- ForceRender: `Dispatcher.Invoke(DispatcherPriority.Render)` で保留中の描画を強制反映。
- ItemsPresenter は FindVisualChild<T> で初回取得後にキャッシュし、2回目以降のオーバーヘッドをゼロにする。

### 13.2 フェード対象外
- × クリアボタンでのリセット：即座に消える操作感が自然。
- タブ切替：Visibility の切替で十分。
- 列ヘッダー：常時表示（検索中も列名が見える）。

## 16. 設定ファイル

### 16.1 file_search_config.json
| 項目 | 必須 | 既定値 | 説明 |
|------|------|--------|------|
| searchFolder | ○ | — | 検索対象フォルダの絶対パス。文字列（単一）または配列（複数）。配列時は横断検索。 |
| depositSeizureListScript | | — | deposit_seizure_list.exe の絶対パス。設定すると「差押リストに追加」が有効。 |
| searchHistoryMax | | 5 | サイドパネルの検索履歴タブに保持する件数の上限。 |
| fileHistoryMax | | 10 | サイドパネルのファイルタブに保持する件数の上限。 |

### 16.2 JSON パース
- LGWAN 環境で外部ライブラリが使えないため、手動パース方式を採用。
- 行単位で `"key":` を検出し、値を抽出する簡易パーサー。
- `_README` 配列はドキュメント用コメントとして無視される。
- 未設定項目はデフォルト値が適用される。

## 18. コンソール版との主な差異

| 項目 | コンソール版（.ps1） | GUI版（.cs/.exe） |
|------|-------------------|------------------|
| 実行環境 | PowerShell 5.0 | .NET Framework 4.0 WPF |
| 検索結果表示 | 番号付きテキストリスト | ListView（4列、ソート可、複数選択対応） |
| デフォルトソート | 更新日 昇順 | 更新日 **降順**（最新ファイルが先頭） |
| 1件時の動作 | 自動選択 → ファイルオープン | 手動選択が必要（GUIの操作モデルに合わせた変更） |
| ファイルオープン後 | メニューループ（再開く/検索/終了） | 常時操作可能（メニューループ不要） |
| 検索履歴 | テキスト表示のみ | サイドパネルタブ（クリック即検索） |
| ファイル履歴 | H コマンド → 番号選択 | サイドパネルタブ（常時表示） |
