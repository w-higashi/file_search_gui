# 預貯金照会結果 検索ツール（file_search）GUI版 機能要件書

## 1. ツールの目的

共有ネットワーク上に保存された預金照会結果ファイル（.xlsm）を、宛名番号で素早く検索し、内容確認から差押予定一覧への追加までをシームレスに行うための補助ツール。deposit_seizure_list.exe との連携により、「ファイルを探す → 内容を確認する → 差押リストに追加する」という一連の業務フローを1つのツール内で完結させる。

本書はWPF GUI版（.cs/.exe）の機能要件を記述する。コンソール版（.ps1）の要件書も併せて参照のこと。

## 5. 検索ボックス

### 5.1 構成
- 虫眼鏡アイコン（左端、#999）+ TextBox（Consolas 13px）+ × クリアボタン（右端）。
- プレースホルダ「宛名番号を入力...」（#999、テキスト入力時に非表示）。
- 右隣に「検索」ボタン（AB スタイル、虫眼鏡アイコン付き）。
- 検索ボタンの右に「★」ボタン（お気に入りフォルダ検索の ON/OFF トグル）。

### 5.2 × クリアボタン
- テキストがあるときのみ表示（TextChanged で Visibility を切替）。
- 丸背景（20px, CornerRadius=10）に × アイコン（Line 2本）。
- 通常: 背景 #E0E0E0 / アイコン #777。ホバー: 背景 #CCC / アイコン #555。
- Focusable=False（クリックしても検索ボックスのフォーカスが外れない）。
- クリック時の動作：検索中なら CancelAsync で中断 → テキストクリア → スピナーオーバーレイ非表示 → フッタークリア（色も #666 に復帰）→ 検索ボタン有効化 → ボタン無効化 → 検索ボックスにフォーカス戻し。
- ★ ON 時は追加動作：テキストクリア後に全件再走査を実行（IsBusy 時は pendingFavoriteRefresh フラグで Completed 後に再走査）。

### 5.3 検索実行
- Enter キーまたは「検索」ボタンクリックで ExecuteSearch を呼び出す。
- 通常モード: 空文字列の場合は無視する。
- ★お気に入りモード: 空文字列で全件表示、テキスト入力時はお気に入りフォルダ内で絞り込み。
- 検索中（BackgroundWorker が IsBusy）は再実行を無視し、検索ボタンも無効化する。

## 9. 履歴機能

### 9.1 検索履歴
- 検索実行時に履歴に追加（重複除去 → 先頭挿入 → 上限超過分を削除）。
- 上限：config.SearchHistoryMax（既定 5、JSON で変更可）。

### 9.2 ファイル履歴
- ファイルオープン時に履歴に追加（重複除去 → 先頭挿入 → 上限超過分を削除）。
- パスとオープン日時（DateTime）のペアで保存。
- 上限：config.FileHistoryMax（既定 10、JSON で変更可）。

### 9.3 永続化
- exe と同じフォルダの `history/` サブフォルダに、`search_{USERNAME}.json` を自動作成する。
- 検索履歴・ファイル履歴・お気に入りフォルダ・lastSeenVersion を1つの JSON ファイルに保存する。
- 検索実行時・ファイルオープン時・お気に入りフォルダ変更時・アップデート告知閉じ時に都度保存する。
- ファイル履歴の復元時は、存在するファイルのみを対象とする（削除済みファイルは自動除外）。
- お気に入りフォルダの復元時は、存在するフォルダのみを対象とする。
- 読み込みまたは書き込みに失敗した場合は、警告を1回表示し、以降はセッション内のみのメモリ保持に切り替える。
- 差押登録ログは別ファイル `history/seizure_log_{USERNAME}.json` に保存される（deposit/insurance ツールが書き出し、file_search が読み込み専用）。

### 9.4 削除操作
| 操作 | 動作 |
|------|------|
| Delete キー | アクティブタブの選択中項目を削除 |
| 右クリック「削除」 | アクティブタブの選択中項目を削除 |
| 右クリック「全て削除」 | アクティブタブの履歴を全件クリア |

## 10. ファイル操作

### 10.1 ファイルを開く
- `Process.Start` で OS の既定アプリケーション（Excel）で開く。`UseShellExecute = true`。
- オープン前に `File.Exists` で存在チェック（不在時は MessageBox 警告）。
- オープン成功後にファイル履歴に追加し、フッター左にファイル名を表示。
- 検索結果とファイル履歴の両方から実行可能（共通の OpenFile メソッドに合流）。

### 10.2 フォルダを開く
- `explorer.exe /select,{fullPath}` で親フォルダを Explorer で開き、該当ファイルを選択状態にする。
- コンテキストメニューから実行可能。

### 10.3 複数選択
- 検索結果テーブルは SelectionMode='Extended'（Ctrl/Shift による複数選択対応）。
- 複数選択時は一括でファイルを開く / 差押リストに追加が可能。

## 11. deposit_seizure_list.exe との連携

### 11.1 呼び出し方式
- `Process.Start` で deposit_seizure_list.exe を別プロセスとして起動する。
- 引数にファイルパスを渡す（複数ファイルの場合はスペース区切り、各パスをダブルクォートで囲む）。
- `--log=` 引数で差押登録ログファイルのパスを先頭に渡す。
- `UseShellExecute = false`、`EnableRaisingEvents = true` で起動し、`Process.Exited` でログ再読み込み（`ReloadSeizureLog`）を実行する。
- `Dispatcher.BeginInvoke` で UI スレッドに戻す。

### 11.2 設定
- file_search_config.json の depositSeizureListScript に .exe のパスを設定する。
- 未設定または該当ファイルなしの場合は、「差押リストに追加」ボタンが非表示となり、検索とファイルを開く操作のみ可能。

### 11.3 プロセス競合チェック
- 「差押リストに追加」実行前に `Process.GetProcessesByName` で deposit_seizure_list が起動中かチェックする。
- プロセス名は config.DepositScript からファイル名を動的取得（`Path.GetFileNameWithoutExtension`）し、ハードコードを回避。
- 起動中の場合は MessageBox で「差押予定一覧作成ツールが起動中です。処理が完了してから再度実行してください。」と表示し、起動を中断する。

### 11.4 エラーハンドリング
- ファイルが見つからない場合：MessageBox 警告。
- プロセス起動失敗：MessageBox エラー。

## 13. 非同期検索

### 13.1 BackgroundWorker パターン
- ファイル検索（EnumerateFiles）はネットワーク共有の再帰走査で時間がかかるため、BackgroundWorker で非同期実行する。
- UI スレッドのブロックを防ぎ、検索中はスピナーオーバーレイを表示する。

### 13.2 処理フロー
| ステップ | スレッド | 内容 |
|---------|---------|------|
| 1 | UI | 検索履歴更新、フェード準備、検索ボタン無効化、オーバーレイ表示 |
| 2 | UI | `searchWorker.RunWorkerAsync(query)` でワーカー起動 |
| 3 | ワーカー | EnumerateFiles + フィルタリング + ソート |
| 4 | UI | 結果反映、UpdateResultList、オーバーレイフェードアウト + データ行フェードイン |

### 13.3 キャンセル
- `WorkerSupportsCancellation = true` で協調的キャンセルを有効化。
- フォルダ単位・ファイル単位で `CancellationPending` をチェック。
- × クリアボタンで `CancelAsync()` を呼び出し、即時中断。

### 13.4 二重実行防止
- `searchWorker.IsBusy` チェックで検索中の再実行を無視。
- 検索ボタンも無効化して UI 上も操作不可。
- 検索履歴タブのクリックも IsBusy 時は無視。

## 14. アニメーション

### 14.1 スピナーオーバーレイ
- ListView と同じ Grid 内に Border で重ねて配置（データ行エリアのみ覆う）。
- 列ヘッダーの下に配置: `FindVisualChild<GridViewHeaderRowPresenter>` で実高さを取得し、`searchOverlay.Margin` に反映。
- 背景: #CCFFFFFF（半透明白）。
- スピナー: 270°弧（Path + ArcSegment `M 14,2 A 12,12 0 1 1 2,14`）、28×28px、#005FB8 青。
- 回転: RotateTransform + DoubleAnimation（By=360, 1秒, Forever）。
- IsVisibleChanged で自動開始/停止（非表示時のリソース消費ゼロ）。

### 14.2 検索結果フェードイン
- 検索完了時に ListView 内の ItemsPresenter（データ行のみ、列ヘッダーは含まない）を対象とする。
- DoubleAnimation(0→1, 100ms) でフェードイン。
- ItemsPresenter は FindVisualChild<T> で初回取得後にキャッシュし、2回目以降のオーバーヘッドをゼロにする。

### 14.3 クロスフェード
- 検索完了時にスピナーオーバーレイのフェードアウト（100ms）とデータ行のフェードイン（100ms）が同時に走る。
- オーバーレイ: Opacity 1→0 → Completed で Visibility=Collapsed。
- 次回表示前に `BeginAnimation(null)` で前回フェードアウトの残値をクリア。

### 14.4 フェード対象外
- × クリアボタンでのリセット：即座に消える操作感が自然。
- タブ切替：Visibility の切替で十分。
- 列ヘッダー：常時表示（検索中も列名が見える）。
- スピナーオーバーレイの表示：即時 Visible（ローディング中であることを即座に伝える）。

### 17.1 file_search_config.json
| 項目 | 必須 | 既定値 | 説明 |
|------|------|--------|------|
| searchFolder | ○ | — | 検索対象フォルダの絶対パス。文字列（単一）または配列（複数）。配列時は横断検索。 |

### 17.2 update_notice.json（任意）
- exe と同じフォルダに配置する。ファイルが存在しない場合は告知をスキップする。
- version: 告知バージョン文字列。lastSeenVersion と比較して異なれば告知を表示する。

- sections: 各 section は heading と items（文字列配列）を持つ。

- LGWAN 環境で外部ライブラリが使えないため、手動パース方式を採用。
- 行単位で `"key":` を検出し、値を抽出する簡易パーサー。
- `_README` 配列はドキュメント用コメントとして無視される。
- 未設定項目はデフォルト値が適用される。

## 20. ★ お気に入りフォルダ検索

### 20.1 概要
- 検索ボタン右の★ボタンで通常モード / お気に入りモードを切替える。
- お気に入りフォルダ（最大5つ）をユーザーが登録し、そのフォルダ群を検索対象にする。
- ★ ON: 空検索で全件表示、テキスト入力で絞り込み。★ OFF: 従来の searchFolder 検索。

### 20.2 ★ ボタン
- 位置: 検索ボタンの右。ControlTemplate でカスタムスタイル。
- ON 時: 青背景 #E8F0FE + 青ボーダー #005FB8。OFF 時: 白背景 + #D0D0D0 ボーダー。
- 左クリック: ON/OFF トグル（初回かつフォルダ未登録時は設定オーバーレイ表示）。
- 右クリック: 設定オーバーレイ表示。
- ToolTip: 「お気に入りフォルダ検索」。

### 20.3 お気に入り設定オーバーレイ
- メインコンテンツの上に暗転背景 + 中央カードで表示（フェード150ms）。
- 固定5スロット表示（登録済み: フォルダアイコン + パス + ✕削除ボタン、空: 番号 + 「未登録」）。
- 追加方法: パス直接入力 TextBox + 参照ボタン（FolderBrowserDialog）+ 追加ボタン（AB スタイル）。
- 右上✕で閉じる。背景クリック / Escape でも閉じる。
- 閉じる時: favFoldersSnapshot との比較で変更があった場合のみ再走査。
- フォルダ0件で閉じた場合: ★を ON にしない。

### 20.4 モード切替時の動作
- ON → テキストクリア → 全件走査開始。OFF → テキストクリア → 結果テーブルクリア。
- フッター右: ★ ON 時は「★ お気に入りフォルダ（N 個）」（#005FB8）。★ OFF 時は searchFolder パス。
- セクションヘッダー: ★ ON 時は「★ お気に入りフォルダ」バッジ表示。

### 20.5 永続化
- `history/search_{USERNAME}.json` の `favoriteFolders` キーに保存。
- 復元時は存在するフォルダのみ対象（最大5件）。

## 21. アップデート告知

### 21.1 概要
- 機能追加時に既存ユーザーへ新機能を案内するための仕組み。
- `update_notice.json`（exe と同じフォルダ、任意）から告知内容を読み込む。
- `lastSeenVersion`（history JSON に保存）と比較し、異なれば初回起動時にオーバーレイ表示。

### 21.2 告知オーバーレイ
- ContentRendered で CheckUpdateNotice を実行（searchBox.Focus の後）。
- 暗転背景 + 中央カード（フェード150ms）。タイトル「アップデートのお知らせ」+ 右上✕。
- バージョンバッジ: pill 形式（#E8F0FE 背景、#005FB8 テキスト、Padding 10,2,10,5）。
- features → sections → items を動的レンダリング（BuildNoticeUI）。
- 閉じる操作: ✕ボタンのみ。背景クリック / Escape では閉じない（見逃し防止）。
- ✕で閉じた時点で `lastSeenVersion` を保存。

### 21.3 エラーハンドリング
- JSON ファイルが存在しない / 読み込み・パース失敗 / version キーがない → 告知スキップ、正常起動。
- 全体を try/catch で囲み、ツール本体の動作に影響しない。
